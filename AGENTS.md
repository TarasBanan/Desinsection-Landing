## 5. Правила видео (критично)

- Всегда: `autoplay muted loop playsinline` на теге `<video>`.
- Всегда: `<source src="hero.webm" type="video/webm">` перед mp4.
- Всегда: `poster="hero-poster.jpg"` на теге `<video>` — это fallback.
- На мобильных (≤768px): отключать видео через JS, показывать poster как background-image — экономия трафика.
- Видео не блокирует LCP. Poster грузится первым.

## 6. Соглашения

- **Именование CSS-классов**: BEM — `.block__element--modifier`.
- **JS**: без классов там, где хватает функций. Без jQuery.
- **Комментарии**: только там, где логика неочевидна.
- **Переменные CSS**: все цвета, шрифты, отступы — только через `var(--*)` из `variables.css`.
- **Нет inline-стилей** в HTML, кроме динамически устанавливаемых через JS.
- **Изображения**: WebP с fallback JPEG. `loading="lazy"` везде, кроме hero-poster.

## 7. Критерии готовности (Definition of Done)

Задача считается выполненной, когда:

- [ ] Lighthouse Performance ≥ 80 (mobile).
- [ ] Нет console.error в браузере.
- [ ] Видео воспроизводится на десктопе; на мобильном показывается poster.
- [ ] Форма отправляется, показывает success-сообщение, не ломается при пустых полях.
- [ ] Все блоки из `CASE.md` присутствуют на странице.
- [ ] Дизайн соответствует `DESIGN.md` (цвета, шрифты, отступы).
- [ ] Страница корректно выглядит на 320px, 768px, 1280px.
