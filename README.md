# Лендинг услуг дезинсекции

Одностраничный mobile-first лендинг для услуги уничтожения постельных клопов. Сайт собран на ванильных `HTML/CSS/JS`, использует видео в hero-блоке на десктопе, заменяет его на poster на мобильных устройствах и ведет пользователя к двум основным целям: расчёту стоимости и отправке заявки.

## Что показывает сайт

Лендинг последовательно проводит пользователя через ключевые блоки:

- фиксированная шапка с логотипом, рейтингом, мессенджерами, CTA и телефоном;
- hero-блок с УТП, быстрыми действиями и отдельной видео-карточкой;
- блок преимуществ;
- онлайн-калькулятор ориентировочной стоимости;
- блок услуг;
- этапы работы;
- пакеты и цены;
- гарантии;
- отзывы;
- FAQ-аккордеон;
- форма заявки;
- футер с контактами.

## Ключевой функционал

- Адаптивный первый экран: на десктопе видео воспроизводится внутри отдельной карточки, на экранах до `768px` видео отключается и остается poster.
- Онлайн-калькулятор: стоимость пересчитывается на лету по площади, типу объекта и степени заражения.
- FAQ-аккордеон: раскрывает один вопрос за раз.
- Форма заявки: проверяет имя и телефон, использует honeypot-поле и показывает статус отправки без перезагрузки страницы.
- Анимации появления: секции и карточки плавно открываются при скролле через `IntersectionObserver`.
- Mobile CTA: на маленьких экранах закреплена панель быстрых действий для звонка и расчёта стоимости.

## Технологии

- `HTML5`
- `CSS3` с разделением по файлам: reset, variables, typography, layout, components, video, animations, responsive
- `Vanilla JavaScript`
- `live-server` для локальной разработки
- `ESLint` и `Stylelint` для проверки кода

## Локальный запуск

1. Установить зависимости:

```bash
npm install
```

2. Запустить локальный сервер:

```bash
npm run dev
```

3. Открыть сайт по адресу:

```text
http://127.0.0.1:5500
```

## Доступные команды

```bash
npm run dev
npm run start
npm run build
npm run lint:js
npm run lint:css
```

`npm run build` в проекте ничего не собирает, потому что это статический лендинг без этапа bundling.

## Поведение по устройствам

- Desktop: видео активно, hero построен в две колонки.
- Tablet и mobile: интерфейс уплотняется, карточки перестраиваются в одну колонку.
- Mobile `<= 768px`: видео останавливается через JS, вместо него остается poster-изображение.

## Ограничения и допущения

- Контент в текущей версии демонстрационный, но структура соответствует сценарию продающего лендинга.
- Отправка формы сейчас имитируется на клиенте через `setTimeout`; интеграция с backend/API не подключена.
- Скриншоты в `docs/screenshots` должны добавляться отдельно.
