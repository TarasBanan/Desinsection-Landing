# Design System — лендинг «Уничтожение постельных клопов»

## 1. Концепция

Дизайн должен быть не «крипто/финтех», а сервисный, доверительный и прикладной: санитарная служба, быстрый выезд, понятная цена, реальные люди, отсутствие визуального шума. Основа — белый фон, светло-серые секции, зелёные сервисные акценты и оранжевые CTA-кнопки. Главный приоритет — мобильная выдача: первый экран должен быстро объяснять услугу, цену/выгоду и давать понятное действие.

Референсы использовать не как копирование, а как источник отдельных решений:
- **Annushka** — вариант первого экрана с видео в отдельном окне справа от текста.
- **Plus Dom** — плотная шапка с рейтингом, мессенджерами, телефоном и зелёно-оранжевой CTA-системой.
- **СЭС Профи** — медицинско-сервисная чистота, доверительный визуальный язык, логотип/шапка для санитарной тематики.
- **UberClean** — зелёный/серый/белый фон, оранжевые кнопки, карточки преимуществ, быстрые действия в hero.

Запрещено использовать стоковые фото/аватары/видео. Визуальный контент — только реальные материалы заказчика: мастер в СИЗ, реальные обработки, реальные отзывы.

## 2. Цветовая палитра

### Основные цвета

| Роль | Цвет | Назначение |
|---|---:|---|
| Primary Green | `#2E8B57` | брендовый зелёный, иконки, статусы, доверительные акценты |
| Deep Green | `#176B43` | hover, тёмные зелёные элементы, активные состояния |
| Light Green | `#EAF6EF` | фоны карточек, бейджи, лёгкие подложки |
| CTA Orange | `#F28C28` | основные кнопки: заявка, расчёт, звонок |
| CTA Orange Hover | `#E67812` | hover/active для CTA |
| Dark Text | `#1F2933` | основной текст |
| Muted Text | `#667085` | подписи, вторичный текст |
| Border Gray | `#D9E1E7` | границы карточек, инпутов, разделителей |
| Soft Gray | `#F5F7F8` | фон секций |
| White | `#FFFFFF` | основной фон |

### Правила цвета

Зелёный — для доверия, сервиса, рейтинга, гарантий, иконок и вторичных кнопок. Оранжевый — только для главных конверсионных действий: «Рассчитать точную стоимость», «Оставить заявку», «Заказать звонок». Серый — для структуры, фонов, бордеров и спокойного сервисного вида. Синий из старого Coinbase-дизайна не использовать.

---

## 3. Типографика

Использовать системный современный sans-serif или подключаемый Google Font. Рекомендуемые варианты: `Inter`, `Manrope`, `Onest`, `Montserrat`. Для санитарной тематики лучше `Manrope` или `Inter`: читаемо, нейтрально, не выглядит «рекламно-дешево».

### Иерархия

| Роль | Mobile | Desktop | Weight | Line-height |
|---|---:|---:|---:|---:|
| H1 | 32–38px | 52–64px | 700–800 | 1.05–1.15 |
| H2 | 26–30px | 40–48px | 700 | 1.15 |
| H3 / card title | 18–22px | 22–26px | 700 | 1.2 |
| Body | 15–16px | 16–18px | 400 | 1.5 |
| Small / caption | 12–14px | 13–14px | 500 | 1.35 |
| Button | 15–16px | 16px | 700 | 1.2 |

Текст в hero должен быть крупным, но не перегруженным. На мобильном не делать длинные строки: ширина текстового блока 100%, заголовок — максимум 4–5 строк.

---

## 4. Layout и сетка

Mobile-first. Базовый контейнер:
- mobile: `padding: 16px`;
- tablet: `padding: 24px`;
- desktop: max-width `1180–1240px`, padding `32px`.

Breakpoints из ТЗ:
- `320px` — минимальная мобильная ширина;
- `768px` — планшет / отключение видео;
- `1280px` — desktop layout.

Вертикальные отступы:
- mobile sections: `48–64px`;
- desktop sections: `80–112px`;
- внутри карточек: `20–28px`.

Радиусы:
- кнопки: `18–999px`, преимущественно pill;
- карточки: `20–28px`;
- видео/hero media: `28–40px`;
- инпуты: `14–18px`.

Тени мягкие, без «пластика»:
- cards: `0 12px 32px rgba(16, 24, 40, 0.08)`;
- floating CTA/header: `0 8px 24px rgba(16, 24, 40, 0.10)`.

---

## 5. Header

Шапка фиксированная, без видеофона. На desktop порядок слева направо:

1. Логотип компании — ориентир по структуре как у **СЭС Профи**.
2. Рейтинг Яндекс Карт: звёзды, оценка, количество отзывов — визуально как у **Plus Dom**.
3. Кнопка «Заказать звонок» — pill, оранжевый или зелёный outline/filled в зависимости от плотности шапки.
4. Мессенджеры **MAX + Telegram** с подписью «Напишите нам» — по логике как у UberClean/Plus Dom.
5. Телефон с подписью «Круглосуточно».

Desktop header допускает 2 строки: верхняя — контакты/рейтинг/CTA, нижняя — навигация. Если места мало, навигацию упростить, а не мельчить контакты.

Mobile header:
- слева логотип;
- справа иконки звонка и Telegram/MAX;
- sticky bottom CTA допускается: «Рассчитать стоимость» + «Позвонить»;
- рейтинг можно вынести компактным бейджем под header или в hero.

Header не должен занимать больше `64–72px` на мобильном.

---

## 6. Hero: два варианта первого экрана

### Вариант A — рекомендуемый

Видео или фото в отдельной карточке справа / ниже текста. Это основной вариант для мобильной выдачи: текст не лежит поверх видео, выше читаемость, ниже риск плохого LCP.

Desktop:
- слева: бейдж/рейтинг, H1, короткое УТП, CTA, вторичная ссылка/телефон;
- справа: карточка с видео или фото мастера в СИЗ;
- под hero: 4 карточки преимуществ.

Mobile:
- сначала H1 и CTA;
- затем карточка с poster-изображением / фото мастера;
- видео на mobile отключается, показывается poster.

Стиль media-карточки:
- radius `28–40px`;
- лёгкая зелёная рамка или декоративная дуга, отсылка к Annushka без прямого копирования;
- поверх можно добавить бейдж «Выезд сегодня» / «Гарантия бесплатно».

### Вариант B — полный видеофон

Видео занимает первый экран под шапкой. Поверх — затемнение `rgba(15, 23, 42, 0.48–0.62)` и текстовый блок.

Использовать только как альтернативу для тестового первого экрана. На mobile видео обязательно заменяется на poster-изображение. Текст должен быть на однотонной затемнённой зоне, а не на «шумной» части кадра.

### Содержимое hero

H1: «Обработка от клопов по разумной цене».

Подзаголовок: коротко раскрыть срочность, гарантию и прозрачную цену. Например: «Приедем в удобное время, обработаем квартиру/дом/офис и заранее рассчитаем стоимость без скрытых доплат».

Primary CTA: «Рассчитать точную стоимость» → якорь на онлайн-калькулятор.

Secondary CTA: «Заказать звонок» или телефон.

---

## 7. Блок преимуществ под hero

4 карточки:
- «Услуги без лишней “воды”»
- «Цена без лишних доплат»
- «Индивидуальный подход»
- «Бесплатная гарантия»

Оформление ближе к UberClean: светлые карточки, иконка в круге, короткий заголовок, 1 строка пояснения. На mobile — сетка 1×4 или 2×2, если текст помещается без сжатия.

---

## 8. Онлайн-калькулятор

Калькулятор — ключевой конверсионный блок. Он должен выглядеть как отдельная большая карточка на светло-сером фоне.

Поля:
- площадь помещения;
- тип объекта: квартира / дом / офис;
- степень заражения;
- контактный телефон после расчёта или перед отправкой заявки.

UI:
- segmented controls / radio cards вместо тяжёлых select;
- результат показывать сразу: «Ориентировочно от X ₽»;
- рядом CTA «Оставить заявку»;
- подпись: «Точная цена подтверждается после уточнения деталей».

На mobile калькулятор должен быть вертикальным, без мелких таблиц.

---

## 9. Секции страницы

### Услуги

Карточки услуг дезинфекции от клопов. Использовать реальные фото или простые линейные иконки. Если реальных фото пока нет — ставить нейтральные placeholder-блоки без стоковых людей.

### Этапы работы

Нумерованные шаги. На desktop можно линия/стрелки, на mobile — вертикальная timeline-сетка:
1. Заявка
2. Уточнение деталей
3. Выезд мастера
4. Обработка
5. Гарантия / рекомендации

### Цены / пакеты

Карточки тарифов. Один пакет выделить как рекомендуемый. Акцент не должен быть агрессивным: зелёная плашка «Рекомендуем» + оранжевая CTA-кнопка. Цена крупно, условия мелко, скрытые доплаты не использовать.

### Гарантии

Спокойный доверительный блок: гарантия, повторная обработка, документы, лицензии/сертификаты. Иконки — щит, документ, чек, календарь.

### Отзывы

3–5 реальных отзывов. Фото клиента использовать только если заказчик предоставляет реальные. Без стоковых аватаров: если фото нет, использовать инициалы в нейтральном круге.

### FAQ

Аккордеон. Вопросы должны быть практичные: безопасность, запах, подготовка квартиры, дети/животные, срок действия, повторная обработка.

### Форма заявки

Поля: имя, телефон, комментарий. Телефон обязательный. Кнопка оранжевая. После отправки — inline success без перезагрузки. Honeypot для спама.

### Footer

Логотип, телефон, email, адрес, политика конфиденциальности, копирайт. Цвет — белый или светло-серый фон, без тяжёлого тёмного футера.

## 10. Компоненты

### CTA buttons

Primary:
- background `#F28C28`;
- hover `#E67812`;
- color `#FFFFFF`;
- radius `999px`;
- padding mobile `14px 18px`, desktop `16px 28px`;
- font-weight `700`.

Secondary:
- background `#EAF6EF`;
- color `#176B43`;
- border `1px solid rgba(46, 139, 87, 0.22)`.

Ghost:
- transparent;
- color `#176B43`;
- underline/arrow допустимы только для текстовых ссылок.

### Cards

Белый фон, radius `24px`, border `1px solid #D9E1E7`, shadow мягкая. Иконки — зелёные/оранжевые, не синие.

### Forms

Инпуты высотой `48–56px`, radius `14–18px`, border `#D9E1E7`, focus `#2E8B57`. Ошибки — спокойный красный `#D92D20`.

### Badges

Рейтинг, гарантия, «выезд сегодня», «без доплат» — pill-бейджи. Фон `#EAF6EF`, текст `#176B43`; для срочных бейджей можно оранжевый фон `#FFF3E6`.

## 11. Видео и изображения

Видео:
- `webm` первым, затем `mp4`;
- `autoplay muted loop playsinline`;
- `poster` обязателен;
- вес видео до `10 МБ`;
- на `≤768px` видео отключается через JS, показывается poster.

Изображения:
- WebP + JPEG fallback;
- `loading="lazy"` везде, кроме hero/poster;
- реальные фото заказчика;
- не использовать stock, AI-generated people, generic disinfecting photos.

Hero-фото мастера в СИЗ должно быть визуально чистым: кадр без мусора, без сильного контраста, с местом под текст/обрезку.

---

## 12. Responsive behavior

Mobile:
- header максимально короткий;
- CTA доступен без скролла;
- hero не должен превращаться в длинную «простыню» до кнопки;
- калькулятор сразу после преимуществ;
- все кликабельные элементы минимум `44px`.

Tablet:
- hero 2 колонки можно включать с `768px`, если хватает ширины;
- карточки 2×2.

Desktop:
- полноценная 2-колоночная hero-сетка;
- преимущества 4 в ряд;
- калькулятор широкая карточка.

---

## 13. Do / Don't

### Do
- Использовать зелёно-серую основу и оранжевый CTA.
- Делать первый экран читаемым на телефоне.
- Показывать реальные фото/видео, а не декоративные стоки.
- Держать карточки преимуществ короткими.
- Визуально отделять header от видеофона.

### Don't
- Не использовать Coinbase-синий и финтех-стилистику.
- Не класть важный текст поверх шумного видео без затемнения.
- Не делать мобильную версию уменьшенной desktop-версией.
- Не перегружать header всеми desktop-элементами на mobile.
- Не использовать стоковые аватары в отзывах.

---

## 14. Prompt guide для генерации интерфейса

Создай mobile-first лендинг для услуги уничтожения постельных клопов. Стиль: белый фон, светло-серые секции, зелёный сервисный акцент, оранжевые CTA-кнопки. Header фиксированный без видеофона: логотип, рейтинг Яндекс, кнопка «Заказать звонок», MAX + Telegram с подписью «Напишите нам», телефон «Круглосуточно». Hero: вариант A — текст слева, справа отдельная карточка с видео/poster или фото мастера в СИЗ; вариант B — альтернативный первый экран с полным видеофоном и затемнением. H1: «Обработка от клопов по разумной цене». CTA: «Рассчитать точную стоимость» ведёт к калькулятору. Под hero — 4 карточки преимуществ: без воды, без доплат, индивидуальный подход, бесплатная гарантия. Без stock-контента; только реальные фото/видео заказчика. UI должен быть чистым, доверительным, конверсионным, адаптивным под 320/768/1280.
